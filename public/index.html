What happens each morning

1
Vercel Cron fires
At 23:00 UTC, Vercel sends an authenticated request to /api/briefing.
2
Fetch yesterday's signups
Queries the Supabase signups table for rows created between yesterday 00:00 and today 00:00 in Singapore time.
3
Summarize with Groq
An LLM prioritizes the signups, groups them by signal, and writes a suggested opener for the highest-priority lead.
4
Deliver the briefing
A styled HTML email goes out via Resend (primary) and Gmail (secondary, optional).
The actual endpoint at /api/briefing is protected by a bearer token. Source on GitHub →
